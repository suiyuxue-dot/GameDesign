# 🎓 研究生求生录 - Graduate Survival Record

## 游戏介绍

基于完整游戏策划案的单页面网页游戏，模拟研究生校园生活，以"与导师斗智斗勇"为核心的卡牌构筑Roguelite游戏。

## 核心玩法

### 游戏机制
- **每周5点精力**：在5种行动中选择分配
- **导师意图推断**：通过线索揭示导师的3个隐藏属性
- **危机事件系统**：随机危机与强制危机，需要策略应对
- **考核节点**：第4/8/12周的三大考核关卡
- **卡牌构筑**：通过道具获取构建不同的流派Build

### 行动类型
1. **完成导师任务**（消耗2精力）：根据导师期望获得满意度
2. **推进论文研究**（消耗2精力）：积累论文进度
3. **调查导师线索**（消耗1精力）：揭示导师隐藏属性
4. **拓展人脉关系**（消耗1精力）：建立人脉网络
5. **休息调整**（消耗0精力）：节省精力

### 导师隐藏属性
- **期望方向**：论文型/平衡型/人脉型
- **压力阈值**：30-50之间的数值
- **奖励偏好**：道具奖励型/声望奖励型/能力奖励型

### 资源系统
- **满意度**：0-100，降至0则退学
- **精力**：每周5点，不跨周累积
- **学术声望**：局内货币，用于获取道具
- **论文进度**：累计推进次数
- **人脉值**：同门互助的基础
- **情报值**：揭示导师属性的关键
- **韧性值**：危机应对能力

### 考核系统
- **期初考核**（第4周）：满意度≥40
- **中期答辩**（第8周）：满意度≥55 且 论文进度≥4次
- **毕业答辩**（第12周）：满意度≥65 且 论文进度≥10次

### 危机事件
- **组会被怼**（1级）：满意度-10
- **紧急改稿令**（2级）：下周精力-1，论文进度-2
- **导师出差失联**（2级）：本周导师任务无效
- **学术不端指控**（3级）：满意度-25
- **表现不佳警告**（1级）：满意度-5

### 道具系统
游戏包含10件核心道具，涵盖5个生态位：
- **引擎类（I-）**：提供持续性被动效果
- **增幅类（A-）**：主动使用提升效率
- **保护类（P-）**：抵挡外部压力
- **终结类（F-）**：高风险高收益
- **特殊类（S-）**：打破常规规则

## 游戏流派

根据策划案设计，游戏支持多种流派构筑：

### 主流派
1. **论文机器**：专注论文产出，中后期自动化满意度维护
2. **人脉达人**：人脉值触发同门互助，降危机+获线索
3. **摸鱼大师**：最小资源维持满意度底线，省出精力触发特殊效果
4. **情报专家**：情报值积累揭示导师意图，精准服务
5. **危机应对专家**：韧性值积累，危机反倒成获益机会
6. **双轨压榨**：叠加精力上限，多线并行

## 使用方法

1. **打开游戏**：直接在浏览器中打开 `graduate-survival.html` 文件
2. **开始游戏**：点击"开始游戏"按钮
3. **每周行动**：选择并执行5种行动之一
4. **应对危机**：当危机触发时，选择应对策略
5. **结束本周**：完成所有行动后点击"结束本周"
6. **通过考核**：在三个考核节点达到要求
7. **完成游戏**：通过毕业答辩或失败退学

## 游戏特色

### 信息不对称
- 导师属性完全隐藏
- 需要通过线索推断
- 线索准确率80%

### 资源压迫
- 每周精力有限
- 需要在导师任务和个人发展间平衡
- 满意度与论文进度双线压力

### 策略深度
- 6条主流派路线
- 25-30种Build组合
- 道具协同效应

## 技术实现

### 技术栈
- 纯HTML + CSS + JavaScript
- 无外部依赖
- 单文件可运行

### 核心算法
- 危机触发率公式：`30% × (1 - 满意度/100 × 0.5)`
- 满意度变化公式：`(匹配度 - 0.33) × 30`
- 随机化与保底机制

## 设计原则

### 不可压缩规则
1. 导师有3个隐藏属性，玩家不可直接看到
2. 满意度每周末结算一次
3. 精力每周刷新，未用不累积
4. 同周不能有超过1个危机
5. 牌组下限4张
6. 卡牌只能通过4个既定途径获取
